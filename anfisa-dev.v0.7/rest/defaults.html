defaults¶

Instance settings

Synopsis¶

dirinfo

Arguments:

ds: optional dataset name

Return value:
{ dictionary
various settings for the instance of Anfisa
}

Description¶

The request returns "hardcoded" constants used by instance of Anfisa:

  • ws.max.count - maximal count of variants in WS-dataset, int, 9000

  • export.max.count - maximal count of variants for export process, int, 9000

  • tab.max.count - maximal count of variants for tab_report, int, 9000

  • ds.name.max.length - maximal length for dataset name, int, 255

  • tag.name.max.length - maximal length for tag name, int, 255

  • sol.name.max.length - maximal length for solution item, int, 255

  • xl.view.count.full - maximal count of variants for full viewing regime in XL-dataset, int, 300

  • xl.view.count.samples.default - count of samples in sampling viewing regime in XL-dataset, int, 25

  • xl.view.count.samples.min - minimal bound for variant count in sampling viewing regime in XL-dataset, int, 10

  • xl.view.count.samples.min - maximal bound for variant count in sampling viewing regime in XL-dataset, int, 150

  • variety.max.rest.size - maximal count of symbols for variety filtering property to provide full statistic, int, 300

  • solution.std.mark - mark for standard solution items, str, "@"

  • run-options - instance run options, list of strings, []

  • run-modes - instance run modes, list of strings, []

  • job-vault-check-period - period of time for check for dataset appearing and changes, in seconds, int, 30

  • if ds argument is set:

    • ds-name - name off dataset (equals to input ds option)

    • can-drop-ds - if the dataset is applicable for drop operation, boolean
